## 🔴 ROOT CAUSE IDENTIFIED

### Database Mismatch - Critical Architecture Bug

**The Problem:**
- **Continuous Backtester Worker** saves data to **Supabase** (PostgreSQL)
- **Research Agent Dashboard** queries from **Local SQLite** (ml-loop.sqlite)
- **Result:** Data goes into Supabase, but dashboard reads from empty SQLite!

```

Research Agent Dashboard

  - signal_snapshots (EMPTY!)
```

## ✅ SOLUTION OPTIONS

### Option 1: Sync Supabase → SQLite (Recommended)
Create a sync worker that copies backtest data from Supabase to local SQLite periodically.

**Pros:**
- Keeps existing architecture
- Fast local queries for dashboard
- Can work offline

**Cons:**
- Added complexity
- Sync lag

### Option 2: Dashboard Reads from Supabase
Update research-agent-dashboard.js to query Supabase instead of SQLite.

**Pros:**
- Single source of truth
- Real-time data
- Simpler architecture

**Cons:**
- Slower queries (network roundtrip)
- Requires Supabase connection

### Option 3: Unified Database
Move everything to Supabase OR everything to SQLite.

**Pros:**
- Clean architecture
- No sync issues

**Cons:**
- Major refactoring
- Migration complexity
