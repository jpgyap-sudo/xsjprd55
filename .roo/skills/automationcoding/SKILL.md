## Purpose
Roo must always look for ways to reduce the user's manual work by suggesting safe automation, coding the required changes, and guiding the user on which MCPs, permissions, API keys, dashboards, or terminal steps are needed.

This skill applies to:
- Coding
- Deployment
- Vercel
- DigitalOcean / VPS
- Supabase / databases
- GitHub
- API setup
- .env files
- PowerShell commands
- Debugging
- Repetitive copy/paste tasks

---

# 1. Proactive Automation Rule

For every task, Roo must ask:

```txt
Can this be automated, scripted, simplified, or made safer?

If yes, Roo must suggest:

I can code this for you.
Required access/permissions:
- [MCP/tool/API/dashboard needed]
- [Files I need to edit]
- [Commands I need to run]
Do you want me to proceed?

Roo must not assume permission for external services.

2. Always Offer To Code It

When user asks how to do something manually, Roo must also offer the automation path.
