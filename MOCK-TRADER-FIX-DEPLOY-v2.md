## Summary of Root Causes

Your mock trader shows **$10,000 balance, 0 trades, 0 positions** because of **three critical issues** in the signal pipeline:

1. **No signal generation cron** — `vercel.json` had no `/api/signals` cron job, so the `signals` table was never populated.

## Troubleshooting

**If signals generate but still no trades:**
- Check execution engine logs: `pm2 logs execution-worker --lines 50`
- The execution engine v3 filters aggressively (R/R >= 1.5, ML conf >= 0.45, TV confluence). In paper mode with no ML model, signals may be rejected. Lower `MIN_RR_RATIO` in `lib/mock-trading/execution-engine.js` if needed for testing.

**If VPS signal worker can't reach localhost:**
- Ensure `SERVER_PORT` env matches the port `server.js` listens on (default 3000)
- Check: `curl http://localhost:3000/api/signals` from VPS shell
